## 业务背景和价值

当前 config 管理员配置表按用户、模型和图片类型配置最大张数，但系统同时将最大值作为新建任务默认值。用户实际常用配置存在个人差异，不适合由管理员逐人维护，且每次手动调整影响操作效率。

本需求通过前端 `localStorage` 记忆用户上一次成功使用的模型与数量配置，用作下次创建任务的默认值，从而：

1. 减少重复调整，提高任务创建效率。
2. 沉淀用户个人常用配置，适配美工和运营的不同使用习惯。
3. 保持 config 仅负责模型可用性和最大张数上限，降低误按最大值生成过多图片的风险。

## 需求概述

在现有“模型与数量配置”模块上增加前端本地缓存，不新增页面、不调整矩阵布局、不修改 config 管理员配置和后端任务接口。

- 同时支持美工版和运营版，两端分别记忆。
- 同一用户、同一业务端内，D1 新增任务和 M1 批量导入共用一份缓存。
- D2 编辑任务仍以任务自身配置为准；只有模型或数量实际变化且保存成功后，才更新个人缓存。

## 功能需求

### 1. 影响范围

| 模块 | 读取缓存 | 更新缓存 | 处理口径 |
|---|---:|---:|---|
| D1 新增任务抽屉 | 是 | 是 | 打开时读取缓存；单个任务配置成功落库后更新缓存。 |
| M1 批量导入弹窗 | 是 | 是 | 与同业务端 D1 共用缓存；本批次至少一个任务成功落库后更新。 |
| D2 编辑任务抽屉 | 否 | 条件更新 | 回填任务自身配置；仅模型或数量实际变化且保存成功后更新。 |
| D3 查看任务抽屉 | 否 | 否 | 保持只读展示。 |
| M2 批量修改 | 否 | 否 | 保持现有“留空不改、0 改为 0、正整数改为指定值”规则。 |
| 复制重跑、批量复制重跑 | 否 | 否 | 继续继承原任务配置，不影响个人缓存。 |

### 2. 缓存内容与隔离

缓存内容包括：

- 启用模型集合。
- 每个启用模型下，当前页面支持的全部图片类型及计划张数。
- 缓存结构版本 `schemaVersion` 或等价字段。

缓存 key 至少按“AI 生图业务命名空间 + 当前用户 ID + 业务端”隔离；美工版与运营版、不同用户之间不得互相读取或覆盖。

每个启用模型必须显式保存全部图片类型，包括值为 `0` 的 cell。字段值为 `0` 表示用户明确不生成，不能因序列化省略而恢复为 config 默认值。未启用模型的历史数量不写入有效数量矩阵，后续重新启用时按当前 config 默认值填充。

本期仅缓存模型与数量配置，不缓存语言、版式、商品资料、SKU/ASIN、appKey、用户信息、Token 或其他业务及鉴权数据。

### 3. 初始化、校验与兼容

D1、M1 每次打开时按以下顺序初始化：

1. 获取当前用户 ID 和当前业务端 config 配置。
2. 查询对应 localStorage 缓存。
3. 缓存合法且版本兼容时，按缓存回显；否则沿用现有默认逻辑：默认仅启用 LinkFox，并按 config 值填充。

读取缓存后需按最新 config 处理：

- 缓存数量不超过 max 时按缓存值回显；超过 max 时自动取当前 max；max 为 `0` 时按 `0` 处理。
- 缓存中启用的模型如果所有图片类型 max 均为 `0`，自动取消启用。
- 对齐后无启用模型时：LinkFox 可用则回退启用 LinkFox 并按 config 填充；LinkFox 也不可用则保持无启用模型，由用户选择。
- 仅按 config 上限对齐，不根据当日剩余额度裁剪；每日额度不足仍沿用现有提交校验。

缓存数量必须为非负整数。缓存无法解析、主体结构或关键字段类型非法、已有 cell 为负数/小数/字符串/空值等，或版本明确不兼容时，整份缓存失效并回退现有默认逻辑。

兼容规则：

- 新增模型：旧缓存继续有效，新模型默认不启用；用户手动启用后按当前 config 填充。
- 新增图片类型：已有有效缓存的用户默认填 `0`，避免系统升级后自动增加计划张数；无历史缓存用户继续沿用系统默认逻辑。
- 历史缓存中已不支持的模型或图片类型直接忽略，不报错。
- 美工版后续增加 A+ 图时按上述新增图片类型规则处理，并在用户下次成功操作后显式写入 A+ 字段，包括值为 `0` 的情况。

当前用户 ID 获取失败时不读不写缓存；config 获取失败时不得单独使用缓存绕过当前权限和上限，继续走页面现有失败处理。

### 4. 写入规则

| 场景 | 更新缓存的成功节点 |
|---|---|
| D1 新增任务 | 单个任务配置成功落库。若任务先保存再提交生图，配置落库后即可更新；后续提交失败不回滚缓存。 |
| M1 批量导入 | 至少一个任务使用本次统一配置成功落库；全部失败时不更新。 |
| D2 编辑任务 | 启用模型集合或数量矩阵相对打开时的任务配置实际变化，且任务配置保存成功。 |

D2 中，仅修改语言、版式、商品资料等非缓存字段，未修改直接保存，或修改后恢复为初始值，均不更新缓存。

localStorage 必须写入本次成功请求实际提交并落库的配置快照：后端返回最终配置时优先使用后端结果，否则使用该次请求快照；不得在成功回调时重新读取可能已经变化的当前表单状态。M1 部分成功时写入本次批量请求使用的统一配置快照。

写入时整份覆盖当前用户、当前业务端缓存，包含启用模型下全部图片类型及值为 `0` 的 cell。校验失败、没有任务成功落库、用户取消/关闭、用户 ID 缺失时不更新。localStorage 写入失败不得影响已经成功的任务主流程。

### 5. 现有按钮与缓存生命周期

- 【按默认填充】：仅按当前 config 重置已启用模型的数量，不改变模型启用状态。
- 【清零】：仅清零已启用模型的数量，不改变模型启用状态。
- 点击按钮本身不更新缓存，后续业务操作成功落库后才更新。
- 本期不新增“使用上次配置”“清除个人记忆”等按钮，也不强制增加缓存提示。

缓存不自动过期，直到被后续成功配置覆盖或用户清除站点数据。退出登录不主动清除；同一用户在同一浏览器重新登录后继续使用。换浏览器、换设备、无痕窗口或清除站点数据后不保留。

同一账号多个标签页以最后一次成功写入为最新缓存；已经打开的 D1/M1 表单不被其他标签页实时覆盖，关闭并重新打开后再读取最新值。

### 6. 本期边界

本期不新增后端用户偏好表，不支持跨浏览器或跨设备同步，不处理历史任务回写；D3、M2、复制重跑和批量复制重跑保持现有逻辑。

## 风险描述

| 风险 | 处理方式 |
|---|---|
| config 上限调整导致缓存超限 | 读取时逐 cell 按最新 max 对齐。 |
| 新增模型或图片类型改变用户原计划 | 新模型默认不启用；已有缓存用户的新增图片类型默认填 0。 |
| 用户配置的 0 被误判为字段缺失 | 启用模型必须显式保存全部图片类型及 0 值。 |
| 异步请求写入错误页面状态 | 使用成功请求快照或后端最终落库结果。 |
| 多用户、多标签页或缓存异常影响使用 | 按用户和业务端隔离；打开表单不被实时覆盖；读写失败不影响主流程。 |

## 验收标准

| # | 验收场景 | 预期结果 |
|---:|---|---|
| 1 | 当前用户当前业务端无缓存，打开 D1/M1 | 默认仅启用 LinkFox，并按当前 config 值填充。 |
| 2 | 在 D1 成功保存配置，再打开同业务端 M1 | M1 回显 D1 上一次成功落库的模型和数量配置。 |
| 3 | M1 至少一个任务成功 / 全部失败 | 至少一个成功时更新为本次统一配置；全部失败时原缓存不变。 |
| 4 | 同一用户在美工版、运营版保存不同配置；同一浏览器切换不同用户 | 业务端之间、用户之间分别读取自己的缓存，不串用。 |
| 5 | D2 打开、修改非缓存字段、未修改保存或修改后恢复原值 | D2 始终回填任务自身配置，且上述情况均不更新个人缓存。 |
| 6 | D2 修改模型或数量并保存成功 | 缓存更新为本次成功落库配置。 |
| 7 | 用户将某图片类型配置为 0 并成功保存后重新打开 | 对应 cell 继续回显 0，不恢复为 config 默认值。 |
| 8 | 点击【按默认填充】或【清零】 | 只调整已启用模型数量，不改变启用模型集合；未落库前不更新缓存。 |
| 9 | 缓存数量超过当前 max，或原启用模型当前全部不可用 | 数量按 max 截断；无启用模型时按 LinkFox 兜底规则处理。 |
| 10 | 新增模型、新增图片类型或美工版新增 A+ 图 | 新模型默认不启用；已有缓存用户新增类型默认填 0，无缓存用户沿用系统默认。 |
| 11 | 缓存损坏、数据类型非法、用户 ID/config 获取失败 | 不使用异常缓存，按对应默认或页面失败逻辑处理，不产生公共缓存 key。 |
| 12 | 保存请求发出后用户继续修改表单，前一请求随后成功 | 缓存写入前一成功请求快照或后端最终落库结果。 |
| 13 | localStorage 写入失败 | 已成功任务不回滚，主流程仍保持成功。 |
| 14 | 标签页 A 已打开 D1，标签页 B 更新缓存 | A 当前内容不被覆盖；重新打开后读取最新缓存。 |
| 15 | 执行 D3、M2、复制重跑或批量复制重跑 | 行为与上线前一致，不读取或更新缓存。 |
| 16 | 当日剩余额度小于缓存计划数量，但各 cell 未超过 max | 页面仍按缓存值回显，提交时沿用现有额度不足校验。 |
