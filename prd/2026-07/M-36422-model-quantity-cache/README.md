## 业务背景和价值

当前 AI 生图系统通过 config 管理员配置表，按业务端、用户、模型和图片类型配置可生成的最大张数。一般同一部门的 appKey 和数量上限保持一致，但系统目前同时将该最大值作为新建任务的默认值。

实际使用时，用户通常不会每次都按最大张数生图，导致每次新增任务或批量导入时都需要重复调整“模型与数量配置”。不同用户的常用模型组合和图片数量存在明显差异，不适合继续扩展 config 管理员配置表，由管理员逐个维护个人偏好，否则会增加配置成本，并混淆“系统上限”和“用户默认偏好”的职责。

本需求通过前端 `localStorage` 记忆每个用户上一次成功使用的模型与数量配置，在下次创建任务时优先回显，从而：

1. 减少用户重复调整模型和生图数量的操作，提高任务创建效率。
2. 沉淀用户个人常用配置，适配不同岗位和人员的实际最佳实践。
3. 保持 config 管理员配置表只负责模型可用性和最大张数上限，避免增加逐人维护成本。
4. 降低用户忘记调整默认最大值而误生成过多图片、消耗额外额度的风险。

## 需求概述

在现有“模型与数量配置”模块基础上增加前端本地缓存逻辑，不新增页面、不调整现有矩阵布局，也不修改 config 管理员配置表和后端任务接口。

本期同时支持：

- 美工版 AI 生图。
- 运营版 AI 生图。

美工版和运营版分别记忆；同一业务端内，新增任务抽屉和批量导入弹窗共用同一份缓存。编辑已有任务时仍以任务自身数据为准；仅当用户实际修改模型或数量配置并保存成功后，才将本次最终配置更新为该用户下一次创建任务的默认偏好。

## 功能需求

### 1. 影响范围

| 模块 | 是否读取缓存 | 是否更新缓存 | 本期处理口径 |
|---|---:|---:|---|
| D1 新增任务抽屉 | 是 | 是 | 打开时优先读取当前用户、当前业务端的缓存；单个任务保存为待提交或保存并提交成功后更新缓存。 |
| M1 批量导入弹窗 | 是 | 是 | 与同一业务端 D1 共用缓存；本批次至少有一个任务成功落库后更新缓存。 |
| D2 编辑任务抽屉 | 否 | 条件更新 | 打开时只回填任务自身配置，不允许缓存覆盖；仅当模型或数量配置实际发生变化且保存成功后更新缓存。 |
| D3 查看任务抽屉 | 否 | 否 | 继续只读展示任务真实配置。 |
| M2 批量修改 | 否 | 否 | 保持“留空不修改、0 改为 0、正整数改为指定值”的现有语义，不参与本次缓存。 |
| config 管理员配置表 | 否 | 否 | 页面、接口和数据结构均不修改。 |

### 2. 缓存内容与隔离

缓存对象为用户本次实际使用的“模型与数量配置”，至少包含：

| 缓存内容 | 说明 |
|---|---|
| 启用模型集合 | 记录用户本次实际启用的模型，例如 LinkFox、自研、阿里绘蛙。 |
| 数量矩阵 | 记录当前启用模型下各图片类型的计划生图张数。未启用模型的历史数量不作为个人默认配置。 |
| 缓存版本 | 用于后续缓存数据结构发生不兼容调整时判断兼容性。 |

缓存 key 必须至少包含：

- 固定的 AI 生图业务命名空间，避免与其他 localStorage 数据冲突。
- 当前登录用户 ID。
- 业务端：美工版 / 运营版。
- 缓存结构版本。

同一用户、同一业务端的 D1、M1、D2 共用同一份缓存；美工版和运营版不得互相读取或覆盖，不同登录用户不得互相读取或覆盖。

本期不缓存以下内容：

- 生成图片语言。
- 卖点图排版版式。
- A+ 图排版版式。
- 该图对应卖点及其他 LinkFox 专属字段。
- SKU、ASIN、商品资料、参考图等任务业务数据。
- appKey、用户名、部门信息、接口响应、Token 或其他鉴权信息。

### 3. 读取时机与默认值优先级

D1 新增任务抽屉或 M1 批量导入弹窗打开并初始化“模型与数量配置”时，按以下顺序处理：

1. 获取当前用户、当前业务端对应的 config 管理员配置，确定当前可用模型、图片类型及每个 cell 的最大张数。
2. 查询当前用户、当前业务端的 localStorage 缓存。
3. 缓存合法且版本兼容时，优先使用缓存中的启用模型集合和数量矩阵。
4. 无缓存、缓存无法解析、数据结构非法或版本不兼容时，整份缓存失效，继续使用现有默认逻辑：默认仅启用 LinkFox，并按当前 config 管理员配置值填充。

D2 编辑任务抽屉打开时，不执行上述缓存读取逻辑，必须以当前任务已保存的启用模型和数量矩阵为准。

D1、M1 仅在抽屉或弹窗每次打开初始化时读取一次缓存。表单打开后，其他标签页更新 localStorage 时，不得实时覆盖当前页面已经回显或正在编辑的内容；关闭并重新打开后再读取最新缓存。

### 4. 缓存版本与字段兼容

缓存版本仅在数据结构发生不向前兼容的重大调整时升级。

以下变化属于兼容性扩展，不应仅因此导致旧缓存整体失效：

- 新增生图模型。
- 新增图片类型。
- 页面不再展示某个历史模型或图片类型。

兼容处理规则：

1. 当前页面新增了旧缓存中不存在的图片类型：旧缓存继续有效，新增图片类型按当前 config 配置值填充。
2. 当前页面新增了旧缓存中不存在的模型：新模型默认不启用；用户手动启用时，按该模型当前 config 配置值填充。
3. 缓存中存在但当前页面已不支持的模型或图片类型：忽略对应缓存数据，不报错。
4. 缓存主体结构无法识别、关键字段类型非法或版本明确不兼容：整份缓存失效并回退现有默认逻辑。

### 5. 缓存值与当前 config 上限对齐

localStorage 只用于记忆个人默认偏好，不能突破当前 config 管理员配置表的限制。读取合法缓存后，需要按当前 config 逐个 cell 校验：

1. 缓存数量小于或等于当前 max：按缓存值回显。
2. 缓存数量大于当前 max：自动按该 cell 当前 max 回显，不标红、不阻断打开。
3. 当前 max 为 0：该 cell 按 0 处理，忽略历史缓存值，并保持现有不可输入状态。
4. 缓存中启用了某模型，但该模型当前所有可展示图片类型的 max 均为 0：自动将该模型调整为未启用。
5. 缓存中启用了某模型，且该模型当前至少有一个图片类型的 max 大于 0：保留缓存中的启用状态；允许该模型部分或全部已配置数量为 0。
6. 缓存中未启用的模型保持未启用；用户后续手动启用时按当前 config 默认值填充，不恢复历史隐藏数量。

读取阶段发生的截断、忽略或自动停用只影响当前页面回显，不立即重写 localStorage；只有后续符合写入条件的业务操作成功后，才按最终有效配置更新缓存。

读取缓存时仅根据 config 中的模型可用性和各图片类型 max 对齐，不根据用户当日剩余额度动态裁剪数量。每日额度不足仍沿用现有提交校验和阻断逻辑。

### 6. 写入时机与变更判断

只有符合以下条件时，才使用本次操作的最终有效配置整份覆盖 localStorage：

| 场景 | 更新缓存的成功节点 |
|---|---|
| D1 新增任务 | 单个任务“保存为待提交”成功，或“保存并提交生图”成功。 |
| M1 批量导入 | 本批次至少有一个任务使用弹窗中的统一配置成功落库。全部任务均未落库时不更新。 |
| D2 编辑任务 | 启用模型集合或数量矩阵实际发生变化，且“保存为待提交”“保存并提交生图”或切换任务触发的自动保存成功。 |

D2 变更判断规则：

1. 以抽屉打开时任务自身的启用模型集合和数量矩阵作为初始值。
2. 用户修改后又恢复为初始值，视为未发生变化，不更新缓存。
3. 仅修改生成图片语言、排版版式、商品资料等非缓存字段时，不更新缓存。
4. 未修改任何内容，直接切换任务触发自动保存时，不更新缓存。

通用写入规则：

1. 写入本次成功操作时最终的启用模型集合和启用模型数量矩阵，不做字段级增量合并。
2. 同一用户、同一业务端以后一次符合条件并成功完成的业务操作结果覆盖前一次缓存。
3. M1 批量导入一次创建多个任务时，只需将弹窗中应用到本批任务的统一配置写入一次。
4. D1 保存失败，或 M1 一个任务都未成功落库时，不更新缓存。
5. 校验失败、接口失败、用户取消或直接关闭抽屉/弹窗时，不更新缓存。
6. 用户仅修改输入值，或仅点击【按默认填充】【清零】但未完成符合条件的成功业务操作时，不更新缓存。
7. localStorage 写入失败不得回滚或阻断已经成功的任务保存、提交或批量导入主流程。

### 7. 现有按钮处理

现有按钮保持原位置和交互，不新增 HTML 原型：

- 页面打开时：优先回显 localStorage 中的上一次成功配置。
- 点击【按默认填充】：仍按当前 config 管理员配置值重新填充，不读取 localStorage。
- 点击【清零】：保持现有清零逻辑。
- 点击【按默认填充】或【清零】本身不更新缓存，后续满足写入条件并成功保存、提交或导入后才更新。

本期不新增“清除个人记忆”“使用上次配置”等按钮，也不强制新增缓存读取或写入成功提示。

### 8. 缓存生命周期与多标签页

1. 缓存不设置自动过期时间，直到被该用户、该业务端后续符合条件的成功配置覆盖，或用户清除浏览器站点数据。
2. 用户退出登录时不主动清除缓存；同一用户在同一浏览器重新登录后继续使用原缓存。
3. 不同用户仍通过用户 ID 隔离，不能互相读取。
4. 同一账号在同一浏览器多个标签页共用 localStorage，以最后一次成功写入的配置为最新缓存。
5. 已经打开的 D1/M1 表单不响应其他标签页的实时缓存变化，不自动覆盖当前输入；重新打开时再读取最新值。

### 9. 美工版 A+ 图兼容

美工版和运营版缓存不得通过固定的 4 类或 5 类图片硬编码实现，应按当前业务端实际返回和展示的图片类型处理。

美工版后续支持标准普通 A+ 图时：

1. 旧版美工缓存中没有 A+ 图字段，不视为缓存异常或版本不兼容。
2. 首次读取旧缓存时，A+ 图数量按当前 config 管理员配置值填充。
3. 用户后续在 D1、M1 或 D2 完成符合写入条件的成功操作后，新缓存需要包含 A+ 图配置。
4. 运营版已有的 A+ 图缓存不与美工版共享。

### 10. config 与 localStorage 职责

| 配置来源 | 职责 |
|---|---|
| config 管理员配置表 | 控制用户可使用的模型、appKey、图片类型可用性及每个图片类型的最大张数上限。 |
| localStorage | 记忆当前用户在当前业务端上一次成功使用的启用模型和数量配置，作为下次创建任务的个人默认值。 |

localStorage 不得替代 config 管理员配置，也不得改变后端额度校验、任务创建、任务编辑和历史任务数据。

## 不在本期范围

1. 不修改 config 管理员配置页面、quota 结构及相关后端接口。
2. 不新增后端用户偏好表，不支持跨浏览器、跨设备或跨域同步个人配置。
3. 不修改 D3 查看任务和 M2 批量修改的现有逻辑。
4. 不缓存 LinkFox 专属字段及商品任务资料。
5. 不新增独立 HTML 页面原型、弹窗、按钮或配置入口。
6. 不处理历史任务数据，不批量回写既有任务配置。
7. 不根据用户当日剩余额度动态修改缓存默认值。

## 异常和边界

1. 用户清除浏览器站点数据、使用无痕窗口、更换浏览器或更换设备后，无法读取原缓存，页面按现有 config 默认逻辑初始化。
2. 同一浏览器登录不同账号时，根据用户 ID 隔离缓存，不能出现账号之间配置串用。
3. localStorage 读取异常、JSON 解析失败或版本不兼容时，不得阻断页面打开，按现有默认逻辑初始化。
4. localStorage 写入失败时，不得回滚或阻断已经成功的任务保存、提交或批量导入主流程。
5. config 管理员配置发生变化后，以最新 config 为准，缓存仅在其允许范围内生效。
6. M1 部分成功时，只要至少一个任务成功落库即更新缓存；全部失败时不更新。
7. 其他标签页更新缓存时，不实时修改当前已打开表单的内容。

## 风险描述

| 风险 | 处理方式 |
|---|---|
| 用户误认为配置可跨设备同步 | 本期明确为浏览器本地缓存，更换设备或浏览器后使用 config 默认值。 |
| 管理员下调模型或图片类型上限 | 读取时逐 cell 按最新 max 截断；模型全部 max 为 0 时自动不启用。 |
| 后续新增模型或图片类型导致旧缓存缺字段 | 新增字段按兼容性扩展处理，不使旧缓存整体失效；新增图片类型按当前 config 值补齐，新模型默认不启用。 |
| D2 浏览或修改其他字段时误覆盖个人偏好 | 仅模型启用状态或数量矩阵实际发生变化且保存成功时更新缓存。 |
| 多账号使用同一浏览器导致缓存串用 | localStorage key 必须包含当前登录用户 ID 和业务端标识。 |
| 多标签页实时覆盖正在编辑的表单 | 表单仅在打开初始化时读取，打开后不响应其他标签页的实时覆盖。 |
| 缓存异常影响任务主流程 | 读取失败回退现有默认逻辑；写入失败不影响主业务操作结果。 |

## 验收标准

| # | 验收场景 | 预期结果 |
|---:|---|---|
| 1 | 当前用户当前业务端无缓存，打开 D1 | 默认仅启用 LinkFox，并按当前 config 管理员配置值填充，保持现有逻辑。 |
| 2 | 在运营版 D1 调整模型和数量后，保存为待提交成功，再打开运营版 M1 | M1 回显 D1 上一次成功保存的启用模型和数量配置。 |
| 3 | 在美工版 M1 调整配置并至少成功导入一个任务，再打开美工版 D1 | D1 回显 M1 本次批量导入使用的配置。 |
| 4 | M1 批量导入部分成功、部分失败 | 至少一个任务成功落库，缓存更新为本次弹窗统一配置。 |
| 5 | M1 批量导入全部失败 | 原缓存保持不变。 |
| 6 | 同一用户分别在美工版、运营版保存不同配置 | 两端分别回显各自缓存，不互相覆盖。 |
| 7 | 同一浏览器切换不同用户账号 | 各账号只能读取自己的缓存，不出现配置串用。 |
| 8 | 用户退出后以同一账号在同一浏览器重新登录 | 继续读取该账号原缓存。 |
| 9 | 已存在缓存，打开 D2 编辑任务 | D2 回填任务自身配置，不被缓存覆盖。 |
| 10 | 在 D2 修改模型或数量配置并保存成功，再打开同业务端 D1 | D1 回显 D2 本次成功保存后的最终配置。 |
| 11 | D2 仅修改生成图片语言、版式或其他非缓存字段并保存成功 | 原模型与数量缓存保持不变。 |
| 12 | D2 未修改模型与数量，直接切换任务触发自动保存 | 原缓存保持不变。 |
| 13 | D2 修改模型与数量后又恢复为打开时原值并保存 | 视为未发生配置变化，原缓存保持不变。 |
| 14 | D1、M1 或 D2 校验失败、接口失败、取消或直接关闭 | 原 localStorage 保持不变。 |
| 15 | 点击【按默认填充】后直接关闭 | 页面按 config 值显示，但原缓存不被覆盖。 |
| 16 | 点击【清零】后直接关闭 | 原缓存不被覆盖。 |
| 17 | 缓存某 cell 为 6，当前 config max 调整为 3 | 下次打开 D1/M1 时该 cell 默认回显 3，不标红、不阻断。 |
| 18 | 缓存启用了某模型，但当前该模型所有图片类型 max 均为 0 | 下次打开时该模型默认不启用。 |
| 19 | 旧缓存中不存在本期新增模型 | 新模型默认不启用；用户手动启用后按当前 config 值填充。 |
| 20 | 当前新增了缓存中不存在的图片类型 | 旧缓存仍可使用；新增图片类型按当前 config 值填充。 |
| 21 | 美工版上线 A+ 图后读取旧版 4 类图片缓存 | 原 4 类配置正常回显，A+ 图按当前 config 值填充。 |
| 22 | 缓存 JSON 损坏、主体结构非法或版本明确不兼容 | 页面不报阻断错误，整份缓存失效并按现有 config 默认逻辑初始化。 |
| 23 | 模拟 localStorage 写入失败，但任务接口已成功 | 任务操作保持成功，不因缓存失败回滚或提示主流程失败。 |
| 24 | 标签页 A 已打开 D1，标签页 B 成功更新缓存 | A 当前内容不被实时覆盖；A 关闭并重新打开后读取 B 的最新配置。 |
| 25 | 当日剩余额度小于缓存计划张数，但各 cell 未超过 config max | 页面仍按缓存值回显；提交时沿用现有额度不足校验。 |
| 26 | 打开 D3 或操作 M2 | D3、M2 行为与本需求上线前一致，不读取或更新本地缓存。 |
