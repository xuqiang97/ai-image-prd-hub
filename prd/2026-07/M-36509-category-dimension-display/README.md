## 业务背景和价值

当前运营版 L2 审核工作台和 L3 沉浸式大屏已经展示商品类目，美工版对应入口暂未展示。本期在美工版相关 L2、L3 入口补充类目，使美工审核和回看图片时可结合商品所属类目判断商品属性及生图内容是否合理，并保持美工版、运营版信息展示口径一致。

同时，美工版和运营版相关 L2、L3 弹窗中的商品尺寸长、宽、高当前无法稳定保持在同一行，“高”会换行至第二行，影响信息阅读和页面整洁度。本期在同批次前端展示调整中一并优化。

## 需求概述

本期不新增页面、弹窗或交互，只调整现有弹窗的信息展示：

1. 美工版三个 L2/L3 入口补充展示类目字段，直接复用运营版对应入口的现有展示规则。
2. 美工版和运营版对应 L2/L3 弹窗的商品尺寸长、宽、高调整为同一行展示。

类目展示仅补充至美工版；商品尺寸布局优化同时覆盖美工版和运营版。

## 功能需求

### 1. 美工版补充展示类目字段

| 业务端 | 页面 | 入口 |
|---|---|---|
| 美工版 | AI 生图 | L2 审核工作台弹窗 |
| 美工版 | AI 生图 | L3 沉浸式大屏弹窗 |
| 美工版 | 查看已完成生图 | L3 沉浸式大屏弹窗 |

以上三个入口均补充展示商品类目，并遵循以下规则：

- 类目字段的数据来源、字段内容、字段名称、展示位置、展示样式、超长处理和空值处理，均与运营版对应入口现有实现保持一致。
- 直接复用系统现有类目数据和运营版展示逻辑，不新增另一套类目字段、取数规则或数据保存逻辑。
- 历史任务和新任务统一按现有类目数据展示；现有数据为空时沿用运营版空值表现，不报错、不影响弹窗正常使用。
- 运营版现有类目展示保持不变；如相关入口使用共享组件，不得出现类目重复展示、位置变化或样式回归。

### 2. 商品尺寸长宽高调整为同一行

商品尺寸布局优化覆盖以下入口：

| 业务端 | 页面 | 入口 |
|---|---|---|
| 美工版 | AI 生图 | L2 审核工作台弹窗 |
| 美工版 | AI 生图 | L3 沉浸式大屏弹窗 |
| 美工版 | 查看已完成生图 | L3 沉浸式大屏弹窗 |
| 运营版 | AI 生图 | L2 审核工作台弹窗 |
| 运营版 | AI 生图 | L3 沉浸式大屏弹窗 |
| 运营版 | 查看已完成生图 | L3 沉浸式大屏弹窗 |

处理规则：

- 将商品尺寸的长、宽、高调整为同一行展示，不再出现“高”单独换行至第二行的情况。
- 仅调整前端布局、间距或容器宽度，不修改长、宽、高字段的数据来源、字段值、展示顺序、单位及现有空值处理。
- 不得通过隐藏、截断某一个尺寸字段或缩减字段内容实现同一行展示。
- 美工版与运营版对应入口的尺寸展示效果保持一致。

### 3. 本期边界

- 不调整 L2、L3 弹窗的审核、查看、图片切换、缩放、任务切换及其他现有操作。
- 不修改类目和商品尺寸的数据定义、取数、保存及历史数据，不进行数据补录或回填。
- 不调整美工版、运营版的列表页、任务创建页、任务编辑页及其他未列明入口。
- 不新增类目或商品尺寸相关配置项、权限及接口调用模式。

## 风险描述

| 风险 | 处理方式 |
|---|---|
| 类目展示范围未覆盖“美工查看已完成生图”的 L3 入口 | 将三个美工版类目展示入口分别列明并逐一验收。 |
| 商品尺寸只调整美工版，未覆盖运营版同类问题 | 尺寸布局同时覆盖美工版、运营版对应的 L2/L3 弹窗，并包含查看已完成生图 L3。 |
| 共享组件调整导致运营版类目重复展示或其他信息布局回归 | 类目仅补充美工版；运营版类目现有表现保持不变，尺寸只做局部布局调整。 |

| # | 验收场景 | 预期结果 |

| 2 | 打开美工版 AI 生图 L3 沉浸式大屏弹窗 | 正常展示商品类目，字段内容及展示效果与运营版对应入口一致。 |
