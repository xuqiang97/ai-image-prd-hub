# PRD 目录说明

本目录用于存放 AI 生图项目的轻量小需求 PRD。

## 存放规则

统一按照月份归档：

```text
prd/YYYY-MM/需求目录/README.md
```

示例：

```text
prd/2026-07/M-36001-localstorage-image-count/README.md
prd/2026-07/TEMP-20260709-localstorage-image-count/README.md
```

## 需求目录命名

优先使用：

```text
M-禅道号-英文短名
```

暂无禅道号时使用：

```text
TEMP-YYYYMMDD-英文短名
```

英文短名建议控制在 3～6 个单词以内，使用小写字母和中划线，便于复制、检索和发给 AI 参考。

## 单需求文档规则

每个需求目录只维护一份 `README.md`，该文件即为唯一正式 PRD。

不再同时维护“GitHub README 实施版”和“钉钉模板版”两份文档，避免重复维护和版本不一致。

## 适用需求

适合放入本目录的需求包括：

- 取数逻辑优化
- 接口字段调整
- 配置项规则
- 状态流转规则
- 前端本地缓存
- 线上问题修复
- 后端处理口径
- 数据同步规则
- 不需要页面原型确认的轻量需求

如需求需要页面交互确认，应单独建立 HTML 原型仓库。
