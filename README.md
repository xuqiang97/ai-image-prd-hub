# AI 生图小需求 PRD 汇总

本仓库用于统一沉淀 AI 生图项目中**不需要单独建立 HTML 原型仓库**的轻量需求 PRD。

## 仓库定位

适用范围包括：取数逻辑优化、接口字段调整、配置项规则、状态流转、前端本地缓存、线上问题修复、后端处理口径、数据同步等小需求。

如需求涉及新增页面、页面改版、复杂弹窗、列表交互、权限可视化、BI 看板等需要页面确认的内容，则应单独建立原型仓库，并通过 **HTML 原型 + README.md PRD** 独立交付。

## 使用方式

1. 按月份在 `prd/YYYY-MM/` 下建立需求目录。
2. 每个小需求独立一个目录，目录名建议使用 `M-禅道号-英文短名`。
3. 每个需求目录内只维护一份 `README.md`，该文件即为唯一正式 PRD。
4. 顶层索引表补充需求记录，方便后续通过仓库地址快速给 AI 参考。
5. 如暂无禅道号，可先使用 `TEMP-YYYYMMDD-英文短名`，后续再按禅道号调整。

## 需求目录命名规范

有禅道号：

```text
M-36001-localstorage-image-count
M-35944-asin-query-logic
M-36012-config-default-value
```

暂无禅道号：

```text
TEMP-20260709-localstorage-image-count
```

## 状态说明

| 状态 | 说明 |
|---|---|
| 草稿 | 需求仍在梳理中，关键规则可能调整 |
| 待评审 | PRD 已成稿，等待业务/研发/测试确认 |
| 已确认 | 需求范围和规则已确认，可进入开发 |
| 开发中 | 研发已开始实现 |
| 已上线 | 功能已发布上线 |
| 已废弃 | 需求不再推进或被其他方案替代 |

## 协作原则

- 本仓库优先服务 AI 生图项目中的轻量小需求。
- README.md 即 PRD，不再拆分 GitHub README 和钉钉模板两份文档。
- PRD 前半部分尽量遵循公司钉钉模板格式，后半部分按实际需求灵活展开。
- 不需要页面原型的需求统一沉淀到本仓库；需要页面交互确认的需求单独建原型仓库。
