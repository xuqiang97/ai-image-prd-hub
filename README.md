# AI 生图轻量需求 PRD Hub

本仓库用于统一沉淀 AI 生图项目中**不需要单独建立 HTML 原型仓库**的轻量需求 PRD。

## 核心原则

> **每个需求最终只维护一份 `README.md`。该 README.md 本身就是唯一正式需求文档，并直接按照公司钉钉模板版 PRD 的格式和风格编写。**

不再额外维护钉钉版、GitHub 实施版、精简版、月份汇总版或模板副本。

需求 README.md 的标题使用一级标题 `#`，后续章节依次使用二级标题 `##`、三级标题 `###`。文档最前部的禅道需求和 GitHub PRD 均直接展示完整 URL，不使用 Markdown 链接文本隐藏地址；GitHub PRD 紧跟在禅道需求下一行，方便复制到钉钉文档后直接查看和回跳原文。

## 适用边界

适合放入本仓库的需求包括：

- 取数逻辑优化
- 接口字段调整
- 已上线接口的兼容扩展
- 配置项规则
- 状态流转
- 前端本地缓存
- 线上问题修复
- 数据同步
- 后端处理口径
- 其他不需要页面原型确认的轻量需求

如需求涉及新增页面、页面改版、复杂弹窗、列表交互、权限可视化、BI 看板或其他需要先确认页面效果的内容，应单独建立原型仓库，通过 **HTML 原型 + 唯一 README.md PRD** 独立交付。

判断标准不是“前端还是后端”，而是：

> **本需求是否需要单独制作 HTML 原型确认。**

## 仓库结构

```text
ai-image-prd-hub/
├── README.md
├── AGENTS.md
└── prd/
    └── YYYY-MM/
        └── M-禅道号-英文短名/
            └── README.md
```

说明：

- 根目录 `README.md`：仓库说明和唯一需求索引。
- 根目录 `AGENTS.md`：指导 ChatGPT、Codex 等 AI 如何生成和维护需求 PRD。
- 每个需求目录下的 `README.md`：该需求唯一正式 PRD，可直接用于研发、测试、钉钉流程和后续 AI 参考。

## 新增需求流程

1. 判断本需求是否需要 HTML 原型。
2. 梳理需求边界、是否拆分和关键缺失信息。
3. 在聊天中完成需求澄清、实现口径和验收标准确认。
4. 等用户提供正式需求名称、禅道编号和禅道链接。
5. 信息齐全后，在 `prd/YYYY-MM/` 下直接建立正式需求目录。
6. 生成该需求唯一的 `README.md` PRD。
7. README.md 前半部分遵循公司钉钉模板，后半部分按需求实际情况灵活展开。
8. 新增需求后，只更新本页的唯一需求索引。

默认不提前创建 `TEMP` 临时目录。没有正式需求名称和禅道信息时，只继续讨论和整理；只有用户明确要求提前落库时，才创建临时目录。

涉及已被下游在线使用的接口时，PRD 必须明确现有调用兼容、字段变更、参数优先级、发布顺序、下游沟通和正式接口文档更新；详细规则以 `AGENTS.md` 为准。

## 原需求更新与新需求判断

以下情况直接更新原需求 README.md：

- 同一个禅道需求尚未进入独立开发，继续补充或修正需求。
- 评审前调整范围、规则、字段或验收口径。
- 研发或测试针对原需求提出澄清，本质仍是同一实施范围。

以下情况应新建需求目录和 README.md：

- 已有新的独立禅道需求。
- 原需求已经上线，本次属于后续迭代。
- 新增内容可以独立开发、测试或上线。
- 本次存在新的需求边界和验收标准。
- 线上问题被单独转为修复需求。

新需求依赖历史需求时，应在唯一 README.md 中增加“关联需求与参考资料”，链接相关历史 PRD、原型仓库或业务资料。

## 需求目录命名

正式需求目录：

```text
M-36001-localstorage-image-count
M-35944-asin-query-logic
```

命名说明：

- 英文短名使用小写字母和中划线。
- 建议控制在 3～6 个单词以内。
- 默认等待正式禅道号确定后直接创建 `M-禅道号-英文短名` 目录。
- 不默认创建 `TEMP` 目录，避免后续删除、迁移和重命名。

## 给 AI / Codex 的使用方式

直接提供本仓库地址和本次需求内容即可。AI 应优先读取：

1. `AGENTS.md`：仓库操作和 PRD 写作规则。
2. 历史需求目录：复用已经确认的业务口径和写法。

推荐提示词：

> 请参考本仓库的 `AGENTS.md` 和历史需求。先完成需求澄清，等我提供正式需求名称、禅道编号和链接后，再创建正式需求目录，生成唯一一份 README.md PRD，并同步更新顶层需求索引。
