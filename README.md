# AI 生图轻量需求 PRD Hub

本仓库用于统一沉淀 AI 生图项目中**不需要单独建立 HTML 原型仓库**的轻量需求 PRD。

## 核心原则

> **每个需求最终只维护一份 `README.md`。该 README.md 本身就是唯一正式需求文档，并直接按照公司钉钉模板版 PRD 的格式和风格编写。**

不存在第二份“钉钉版 PRD”，也不额外维护单独的实施版、精简版或模板副本。

## 适用边界

适合放入本仓库的需求包括：

- 取数逻辑优化
- 接口字段调整
- 配置项规则
- 状态流转
- 前端本地缓存
- 线上问题修复
- 数据同步
- 后端处理口径
- 其他不需要页面原型确认的轻量需求

如需求涉及新增页面、页面改版、复杂弹窗、列表交互、权限可视化、BI 看板或其他需要先确认页面效果的内容，应单独建立原型仓库，通过 **HTML 原型 + 唯一 README.md PRD** 独立交付。

判断标准不是“前端还是后端”，而是：

> **本需求是否需要单独制作 HTML 原型确认。**

## 仓库结构

```text
ai-image-prd-hub/
├── README.md
├── AGENTS.md
└── prd/
    └── YYYY-MM/
        └── M-禅道号-英文短名/
            └── README.md
```

说明：

- 根目录 `README.md`：仓库说明和唯一需求索引。
- 根目录 `AGENTS.md`：指导 ChatGPT、Codex 等 AI 如何生成和维护需求 PRD。
- 每个需求目录下的 `README.md`：该需求唯一正式 PRD，可直接用于研发、测试、钉钉流程和后续 AI 参考。

## 新增需求流程

1. 先判断本需求是否需要 HTML 原型。
2. 不需要原型时，在 `prd/YYYY-MM/` 下建立独立需求目录。
3. 直接生成该需求唯一的 `README.md` PRD。
4. README.md 前半部分遵循公司钉钉模板，后半部分按需求实际情况灵活展开。
5. 新增需求后，只需同步更新本页的需求索引。

## 需求目录命名

有禅道号：

```text
M-36001-localstorage-image-count
M-35944-asin-query-logic
```

暂无禅道号：

```text
TEMP-20260710-localstorage-image-count
```

命名说明：

- 英文短名建议使用小写字母和中划线。
- 建议控制在 3～6 个单词以内。
- 禅道号确定后，可将 `TEMP` 目录调整为正式目录名。

## 给 AI / Codex 的使用方式

直接提供本仓库地址和本次需求内容即可。AI 应优先读取：

1. `AGENTS.md`：仓库操作和 PRD 写作规则。
2. 历史需求目录：复用已经确认的业务口径和写法。

推荐提示词：

> 请参考本仓库的 `AGENTS.md` 和历史需求。本次需求不需要 HTML 原型，请在对应需求目录中直接生成唯一一份 README.md PRD。该 README.md 本身就是按照公司钉钉模板编写的正式需求文档，并同步更新顶层需求索引。
